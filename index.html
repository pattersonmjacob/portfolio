How I build learning

Clean process, no theater. Define the performance goal, design the moments of decision, ship fast, then measure what changed.

Design principles

  • Start with the behavior, not the content. If the learner can’t do the thing at the end, the module failed.
  • Lower cognitive load. Fewer words, stronger structure, and feedback that actually teaches.
  • Respect the learner. Clear pace, purposeful interactions, and no busywork clicks.

Typical deliverables

  • Front-end analysis, audience + context, constraints, success metrics
  • Storyboards, scripts, decision maps, and prototype screens
  • SCORM / web build, accessibility pass, QA, publish + handoff
  • Evaluation plan: completion, accuracy, time-to-competency, field outcomes

Tools and capabilities

Tools are just tools. The real value is translating messy business needs into learning people can use.

Build and ship

  • Articulate Storyline / Rise 360 (SCORM packaging, interactions, branching)
  • Web learning (HTML/CSS/JS shells, interactive prototypes)
  • Figma (flows, components, design systems)

Strategy and measurement

  • Performance-based objectives, rubrics, and assessment design
  • Learning analytics (completion, confidence lift, accuracy, time-to-competency)
  • Stakeholder alignment, narrative building, and exec-ready summaries
